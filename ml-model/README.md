# Housing Price Prediction API

A production-ready REST API that predicts residential property prices using a Ridge
regression model trained on 7 structural and location features.

**Model performance:** R² = 0.9857 | RMSE = $8,949 | MAE = $8,629

---

## Project Structure

```
housing-api/
├── app/
│   ├── __init__.py
│   ├── main.py             # FastAPI app — all endpoints + middleware

```

## Quickstart

### Option A — Docker (recommended)

```bash
# Build image (trains model automatically during build)
docker build -t housing-api .

# Run single container
docker run -p 8000:8000 housing-api

# Run with nginx load balancer + 4 replicas
docker compose up --scale api=4
```

### Option B — Local (dev)

```bash
pip install -r requirements.txt

# Train the model first
python train.py

# Start the API
uvicorn app.main:app --reload --port 8000
```

## API Endpoints

### `GET /health`
Liveness check. Safe to poll at high frequency (no model inference).

```bash
curl http://localhost:8000/health
```
```json
{"status": "ok", "model_loaded": true, "model_type": "Ridge Regression"}
```

### `POST /predict`

#### Single prediction

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "features": {
      "square_footage": 1700,
      "bedrooms": 3,
      "bathrooms": 2.0,
      "year_built": 2005,
      "lot_size": 5500,
      "distance_to_city_center": 4.1,
      "school_rating": 7.8
    }
  }'
```
```json
{"predicted_price": 275000.0, "currency": "USD"}
```

#### Batch prediction

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "features": [
      {"square_footage": 1700, "bedrooms": 3, "bathrooms": 2.0,
       "year_built": 2005, "lot_size": 5500,
       "distance_to_city_center": 4.1, "school_rating": 7.8},
      {"square_footage": 2200, "bedrooms": 4, "bathrooms": 3.0,
       "year_built": 2010, "lot_size": 7000,
       "distance_to_city_center": 7.0, "school_rating": 8.8}
    ]
  }'
```
```json
{"predictions": [275000.0, 361237.0], "count": 2, "currency": "USD"}
```

## Performance Architecture

| Technique | What it solves |
|---|---|
| Model loaded at startup (lifespan) | Zero disk I/O per request |
| Numpy batch matrix multiply | No Python loops — handles 10k rows as fast as 1 |
| `asyncio.run_in_executor` + ThreadPoolExecutor | CPU-bound sklearn doesn't block the async event loop |
| Pydantic v2 validation | Bad payloads rejected before touching the model |
| GZipMiddleware | Compresses large batch responses |
| Gunicorn + Uvicorn workers (4×) | Multi-process; each handles concurrent async requests |
| Nginx `least_conn` upstream | Routes to replica with fewest active connections |
| Horizontal scaling (`--scale api=N`) | Stateless containers — no shared state needed |
| `/health` has no model invocation | Load balancer can poll at high frequency for free |

## Features

| Feature | Type | Description |
|---|---|---|
| `square_footage` | float | Total living area (sq ft) |
| `bedrooms` | int | Number of bedrooms (1–20) |
| `bathrooms` | float | Number of bathrooms (0.5–20) |
| `year_built` | int | Year of construction (1800–2025) |
| `lot_size` | float | Lot size (sq ft) |
| `distance_to_city_center` | float | Distance in miles |
| `school_rating` | float | Local school rating (0–10) |

## Model Details

- **Algorithm:** Ridge Regression (L2 regularisation, α=1.0)
- **Preprocessing:** StandardScaler (zero mean, unit variance)
- **Pipeline:** `sklearn.pipeline.Pipeline` — scaler + model as single joblib artifact
- **Validation:** 80/20 train-test split + 5-fold cross-validation
- **Serialization:** `joblib.dump(..., compress=3)` — efficient numpy array storage
